# 약관동의 중간발표 5분 스크립트

## 1페이지: 제목

안녕하세요. 저희 팀은 **약관 분석 AI 서비스, 약관동의**의 개발 현황을 발표하겠습니다.

저희 서비스는 사용자가 약관을 직접 읽고 판단하기 어려운 문제를 해결하기 위해, 약관 내용을 AI가 분석하고 위험 조항을 쉽게 설명해주는 서비스입니다.

현재 기준으로 백엔드 P1 핵심 기능은 대부분 구현 완료된 상태이고, 핵심 모듈 5개와 주요 분석 기능을 중심으로 프론트엔드 연동과 품질 검증을 진행하고 있습니다.

## 2페이지: P1 핵심 개발 현황 1

먼저 P1 핵심 개발 현황입니다.

현재 전체 개발 항목은 총 13개로 정리했고, 백엔드 기준으로는 주요 기능 구현을 완료한 상태입니다. 남은 작업은 프론트 화면 연동과 실제 샘플 기반 품질 검증 중심입니다.

완료된 기능부터 말씀드리면, 먼저 **URL 입력 분석 기능**이 구현되어 있습니다. 사용자가 약관 페이지 URL을 입력하면, 웹 페이지를 크롤링해서 텍스트를 추출하고 분석까지 진행할 수 있습니다.

그리고 URL 분석 과정에서 발생할 수 있는 404, 접근 불가, 타임아웃 같은 오류도 사용자에게 적절한 메시지로 반환하도록 처리했습니다.

또한 **PDF 업로드 기능**도 구현했습니다. 텍스트 기반 PDF는 PyMuPDF로 내용을 추출하고, 스캔 PDF처럼 텍스트가 직접 추출되지 않는 경우에는 페이지를 이미지로 렌더링한 뒤 OCR을 수행하도록 보강했습니다.

텍스트를 직접 붙여넣는 방식도 지원합니다. 사용자가 약관 전문을 입력하면 최소 글자 수를 검사한 뒤 분석이 시작됩니다.

그리고 로그인 없이도 분석할 수 있도록 `session_key` 기반의 비로그인 분석 구조를 적용했습니다.

분석 결과에서는 위험 조항을 HIGH, MEDIUM, LOW로 분류하고, 전체 위험도를 0점에서 100점 기준으로 계산해서 보여줍니다.

또한 위험 문장을 단순히 표시하는 것에서 끝나지 않고, 사용자가 이해하기 쉽게 요약 설명을 생성하는 기능도 구현되어 있습니다.

분석 시작 처리도 비동기 방식으로 개선했습니다. 사용자가 텍스트, URL, 파일 분석을 요청하면 서버는 먼저 `job_id`를 즉시 반환하고, 실제 분석은 백그라운드에서 진행됩니다. 프론트엔드는 이 `job_id`로 결과를 폴링해서 조회할 수 있습니다.

그리고 **이미지 업로드 OCR 기능**도 구현했습니다. JPG, PNG 같은 이미지 파일에서 Tesseract OCR로 약관 텍스트를 추출한 뒤, 동일한 분석 파이프라인으로 연결합니다. Docker 환경에는 Tesseract와 한국어 OCR 데이터 설치 설정도 반영했습니다.

## 3페이지: P1 핵심 개발 현황 2

다음은 추가 핵심 기능입니다.

먼저 **심결례 추천 기능**은 완료되었습니다. 분석된 조항과 관련 있는 공정위 심결례를 RAG와 FAISS 검색을 통해 추천하는 구조입니다.

이 기능을 통해 단순히 “위험하다”라고 판단하는 것이 아니라, 유사한 실제 사례를 함께 제공해서 분석 결과의 근거를 보강할 수 있습니다.

심결례 추천 품질을 높이기 위해 **유사도 임계값 필터링**도 적용했습니다. 검색된 결과 중 기준 점수보다 낮은 결과는 제외하고, 관련성이 높은 심결례만 프론트엔드에 제공하도록 구성했습니다.

마지막으로 **API 에러 응답 구조 통일**은 완료했습니다. 400, 404, 422, 500 같은 오류 응답을 일정한 형식으로 정리해서 프론트엔드에서 일관되게 처리할 수 있도록 했습니다.

## 4페이지: 프론트 진행 상황

다음은 프론트엔드 진행 상황입니다.

현재 v1 기준으로는 **홈 화면과 분석 화면 개발이 완료**된 상태입니다. 홈 화면에서는 사용자가 URL, PDF, 텍스트 방식으로 약관을 입력할 수 있고, 분석 화면에서는 서비스별 위험도와 조항 분석 결과를 확인할 수 있도록 구성했습니다.

이후 개발 일정은 네 가지입니다.

첫 번째는 **분석 페이지 완성**입니다. 마우스 호버 시 AI 요약을 노출하고, 조항별 드롭다운을 통해 상세 내용을 확인할 수 있도록 UI를 마무리할 예정입니다.

두 번째는 **v2 디자인 반영**입니다. 보관함과 설정 화면에 새 디자인을 반영하고 개발할 계획입니다.

세 번째는 **결과 시각화**입니다. 위험도 점수를 0점에서 100점 기준의 게이지 차트로 보여주고, HIGH, MEDIUM, LOW 등급에 따라 색상 하이라이트를 적용할 예정입니다.

네 번째는 **리포트 기능 연동**입니다. 백엔드에는 분석 결과를 PDF로 생성하고 다운로드하는 API가 구현되어 있으며, 이후 프론트엔드에서 다운로드 버튼과 연결할 예정입니다.

## 5페이지: 기술 아키텍처와 API 구조

다음은 기술 아키텍처와 API 응답 구조입니다.

핵심 API 엔드포인트는 분석 입력 방식에 따라 `POST /api/analyze/text`, `POST /api/analyze/url`, `POST /api/analyze/file`로 구성했습니다. 사용자가 약관을 입력하면 서버는 먼저 `job_id`를 반환하고, 분석 완료 후 `GET /api/result/{job_id}`로 결과를 조회할 수 있습니다.

응답에는 전체 조항 수, 고위험 조항 수, 중위험 조항 수, 저위험 조항 수, 전체 위험 비율이 포함됩니다. 각 조항별로는 제목, 위험도, 관련 심결례 정보가 함께 제공됩니다.

전체 데이터 흐름은 다음과 같습니다.

먼저 사용자가 URL, PDF, 이미지, 텍스트 중 하나의 방식으로 약관을 입력합니다.
그다음 전처리 단계에서 텍스트를 추출하고, 분석 단계에서 위험도를 분류합니다.
이후 RAG 검색을 통해 관련 판례나 심결례를 찾고, 최종적으로 JSON 결과를 프론트엔드에 응답합니다.

이 구조를 통해 프론트엔드는 분석 결과를 화면에 바로 시각화할 수 있고, 백엔드는 분석 로직과 데이터 처리를 담당하는 형태로 역할을 분리했습니다.

## 6페이지: 마무리

정리하겠습니다.

현재 약관동의 서비스는 핵심 분석 기능 대부분이 구현된 상태입니다.
URL 분석, PDF 분석, 이미지 OCR, 텍스트 분석, 비동기 분석 시작, 위험도 분류, 심결례 추천, 쉬운 요약, API 응답 구조, PDF 리포트 생성까지 주요 백엔드 기능은 동작하고 있습니다.

앞으로는 프론트 상세 UI, 결과 시각화, 리포트 다운로드 버튼 연동, 실제 약관 이미지와 스캔 PDF 기반 OCR 품질 검증을 중심으로 완성도를 높일 예정입니다.

이상으로 약관 분석 AI 서비스 약관동의의 개발 현황 발표를 마치겠습니다. 감사합니다.
